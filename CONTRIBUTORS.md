helped, code or otherwise).

and [Editorial Policy](https://www.mliphub.com/policy) for our review process.

## How to get listed

### As a contributor

1. Fork the repo and open a pull request editing this file (`CONTRIBUTORS.md`).
2. Add a row under the **Contributors** table with your name, role (code,
   docs, curation, design, QA, model suggestion, etc.), and a link to your
   PR or issue.
3. Release notes will call out contributors added in each version.

### As a citation author

Citation authors are core maintainers or curators for a release. To be added:

1. Open a pull request updating **both**:
   - [`CITATION.cff`](./CITATION.cff) — `authors:` block, with ORCID if available.
   - [`.zenodo.json`](./.zenodo.json) — `creators` array (authoritative for Zenodo).
2. Add yourself to the **Citation authors** table above.
3. The change takes effect in the next tagged release.

## Recognition

MLIP Hub follows the spirit of the
[All Contributors](https://allcontributors.org/) specification — **non-code
contributions count**. Curation, review, documentation, model suggestions,
and bug reports are all first-class contributions.

Questions? Email `support@mliphub.com` or open an issue.
